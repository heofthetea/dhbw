> Run a [[Shell]] command on the target system by injecting it into a vulnerable binary

- Is possible when applications pass unsanitized User Input to a shell

> [!hint] Somewhat the [[Betriebssystem|OS]]-level version of [[SQL injection]]

> [!warning] The only **useful** thing this can do is spawn a [[Reverse Shell]]. (Which already is a lot but after all requires you to do a lot of stuff yourself)
