> Möglichkeiten, in die [[Cloud]] zu migrieren

## Rehost
- die gesamte [[Virtual Machine|VM]] mit der Anwendung in das Datacenter verlagert

> [!hint] In Praxis wird meistens die VM nicht umgezogen, sondern exakt gleich neu installiert.
## Replatform
- Runtimes oder [[DBMS]] ersetzen
	- oft: Weg von Lizensierten sachen wie oracle

## Refactor
- technisches refactoring ([[Containerization]], Server ändern)
- fachliches Refactoring (bsp. Monolith in Microservices zerlegen)

> [!hint] Kann ziemlich lange dauern.
## Repurchase
- Anders einkaufen (bspw. von lifelong lizens per server zu subscription)
